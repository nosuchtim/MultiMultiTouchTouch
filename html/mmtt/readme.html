README for MultiMultiTouchTouch 2.0

The MMTT (MultiMultiTouchTouch) program uses a 3D camera (such as the Creative Senz3D or the Microsoft Kinect) to create multiple 3D multitouch regions in mid-air that you can interact with by merely waving your hands (or other objects) within regions that have been defined. The Space Palette instrument is one of its applications, you can find YouTube videos and and more details at http://spacepalette.com. This document describes how to use MMTT.

Installation

If you're reading this document, you've already installed the MMTT package. If this is the first time you've installed it, you may need to install a few other things, for which you'll find shortcuts in the "All Programs -> Nosuch Media -> MultiMultiTouchTouch" directory. These shortcuts are:
  1. Install Kinect Runtime
  2. Install Kinect SDK
  3. Install loopMIDI
  4. Install Processing
  5. Install Senz3D Runtime
  6. Install Senz3D SDK

Configuration Files

In the "Documents" directory of your Windows login (i.e. %USERPROFILE%\Documents ), you will find a sub-directory named "Nosuch Media/MultiMultiTouchTouch/config". The file "mmtt.json" there controls the default behaviour of MMTT. Edit that file with the text editor of your choice, and adjust the values within it to reflect your desires. The most important values to adjust are "camera" and "patch". The "patch" value should be the name of a file within the "Nosuch Media/MultiMultiTouchTouch/config/mmtt" directory.

Starting MMTT

Use the "Start MMTT" shortcut to start the program. If you don't see a window, you should see an error popup that explains the reason (e.g. "No camera detected"). If you don't see any window or errors, use the "Monitor Debug Output" shortcut to see if there are any errors in the debug logfile, or just look at the file "Nosuch Media/MultiMultiTouchTouch/log/mmtt.debug" in your Documents directory.

Controlling MMTT

When MMTT is running, you interact with it by pointing your browser at http://127.0.0.1:4444.

Usage Scenarios

The 3D multitouch regions that MMTT understands exist within a single plane, with the Z direction pointing at the 3D camera. To define the regions, you can either use a physical frame (any flat surface with large holes) or explicitly specify the region coordinates in the config file. MMTT was primarily designed to be used with a physical frame (like the Space Palette), which simplifies the registration of the regions as well as providing visual guidance for both the player and audience.

Using a physical frame to define the regions

To use a physical frame, first you need to build one. One easy and quick way is to cut holes in a sheet of foam core board. Anything that is flat and has large open regions will do. The open regions don't have to be rectangular, but should be at least a few inches high and/or wide. The parts of the frame between the open regions should be at least 1/2" wide, so that the 3D camera can see them.

Using a physical frame requires a one-time operation that "registers" the regions of the frame, determining the coordinates of the regions.

  1. Put the frame in front of the 3D camera.
  2. Start the MMTT program
  3. Go to the "MMTT Calibration (Advanced)" page in the browser-based interface.
  4. Click on "STOP sending TUIO (Shows Raw Depth)"
  5. Position the frame so that it's centered in the image.
  6. Click on "START sending TUIO". The image may disappear until you adjust the depth values in the next step.
  7. Adjust the parameters of the depth thresholding, which are listed as:
    • Depth for Top of Detection Plane
    • Depth for Bottom of Detection Plane
    • Depth for Mask Alignment
    These parameters identify the distance from the 3D camera to wherever the frame (Mask Alignment) and your hands (Detection Plane) will be. The goal is to set the Depth for Top/Bottom of Detection Plane so that your flat frame is included, but things beyond the plane of the frame (like your body) are not included. The reason there are Top and Bottom values is to allow some skew correction in the vertical direction; the 3D camera will usually be centered horizontally with respect to the frame, but may not be centered vertically.
  8. Start the manual registration process with the "New Registration Start" button on the browser-based user interface. Then, stick your hand into each hole of the frame that you want to use, in order. When you pull your hand out of each hole, you should see (in the image on the screen) the color of that region change to the next color in the color sequence. The registration process concludes either when you stick your hand into the first region for a second time, or when you press the "New Registration Stop" button.
  9. Type a new configuration name into the text field next to the "Save Patch" button, and press "Save Patch". This will save two files - a .json file with the current control parameters, and a .ppm (portable pix map) image file with the current region map that you've just defined.

After registering a particular frame and saving its configuration, you can subsequently just start the MMTT program and load the saved configuration. If you move the 3D camera or the frame, you may have to tweak the threshold parameters and/or re-register and re-save the configuration. If things have only moved slightly, you should be able to just press the "RePoke" button at the top of the browser-based user interface, which pretends to re-register by sticking a "virtual hand" in the center of each of the currently-configured regions. After doing this, you should save the configuration again.

Using a specially-color image to define the regions

If you look at the image file reference.ppm (which is a Portable Pixmap image), you will see the type of colored image that can be used to configure MMTT without having a physical frame. Each different color identifies a different region. White is used to identify the frame (if it is present), which gets ignored when looking for your hands. This Color Reference Page shows you the different colors you should use in this image for each region. If you create a .ppm file containing an image with these colors, you can then load it using the browser-based user interface.

Debugging

Debugging and status information can be viewed with debugview. The browser-based user interface contains a number of controls toward the bottom of the page that can be used to see what's going on. For example, the "Show Region Map" button will let you see the current map of regions. The "Show Raw Depth" button will show you the raw depth image with no thresholding.

OSC / TUIO output

TUIO is a standard for multitouch events using OSC. By default, the TUIO output of MMTT gets sent to port 3333 of host 127.0.0.1, using the profile for 25D surfaces (i.e. the OSC messages start with /tuio/25Dblb). You can change this by using the -n option, letting you can specify one or more semicolon-separated host:port values. For example, -n127.0.0.1:3333;127.0.0.1:7777 would tell MMTT to send TUIO to two clients on host 127.0.0.1, one at port 3333 and one at port 7777.

If you want to interact with programs that expect the TUIO 2D profile, you can tell MMTT to use the 2D profile by using the -o option. For example, -o127.0.0.1:3333 would tell MMTT to send TUIO messages using the 2D profile to a clients on host 127.0.0.1 at port 3333.

The TUIO session id's indicate the region of the frame by multiplying the region id by 1000 and adding it to the session id. So, if you see session id's 2000, 2001, 5000, and 5001, you know that 4 hands have been detected - 2 in region 2 and 2 in region 5.

Command-line Options

Note: there is no space between an option letter and its value (when it has a value).

-fDisable auto-flushing of debug output
-hhtml-directorySpecify the directory from which to serve HTML
-llogfileSpecify the debug log filename
-LDisable debugging to log file
-DDisable all debug output
-ohost:portSpecify the host:port to receive 2D TUIO messages (multiple values can be separated by semicolons)
-nhost:portSpecify the host:port to receive 25D TUIO messages (multiple values can be separated by semicolons)
